# Session Context Folder

Purpose: persist concise context per conversation/session so follow-up sessions can resume quickly.

## Rules
- Create one file per conversation using: `YYYY-MM-DD-session-XX.md`.
- Keep each file short and factual.
- Update `context/INDEX.md` with every new file.
- Include: goals, decisions, changes, validation, and next steps.

## Quick Start
1. Copy `context/CONVERSATION_TEMPLATE.md`.
2. Rename with current date/session number.
3. Fill only what changed in that conversation.
